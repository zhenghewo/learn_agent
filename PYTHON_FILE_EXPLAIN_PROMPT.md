# Python 文件学习解释 Prompt


```text
我正在学习一个 Python 项目。请你帮我解释这个 .py 文件，要求按“初学者但想培养程序员阅读习惯”的方式讲。

解释时请遵守：

1. 先整体说明这个文件的作用
   - 它在项目链路里负责什么
   - 它是入口、工具类、执行器、agent、graph，还是数据结构定义

2. 再按熟练程序员的阅读顺序讲
   - 这个类/函数的职责是什么
   - 它保存了哪些长期状态
   - 它的输入是什么
   - 它的输出是什么
   - 主流程交给谁执行
   - 哪些字段影响控制流
   - 哪些只是占位或细节，先知道大概即可

3. 如果有类，请重点解释：
   - __init__ 里初始化了什么
   - 这些 self.xxx 分别代表什么
   - 哪些对象是长期复用的
   - 哪些数据是每次调用临时创建的

4. 如果有函数，请重点解释：
   - 参数怎么传进去
   - 返回值是什么
   - 函数内部每一步在做什么
   - 哪些变量只是“换了一个名字接住传入的数据”

5. 如果有 dict / TypedDict / dataclass / list / tuple，请单独解释数据结构
   - 这个数据长什么样
   - key 是什么
   - value 是什么
   - 运行过程中它怎么变化

6. 如果有流程控制，请画成文字流程
   例如：
   用户输入
   -> 生成 SQL
   -> 执行 SQL
   -> 如果 error 就 retry
   -> 如果成功就 analyze
   -> 返回 state

7. 如果有 state，请重点解释：
   - state 是整条流程上传递的总状态
   - 哪些字段一开始是空值
   - 哪些字段会被后续节点填充
   - state.update 或字段赋值到底改变了什么

8. 如果有 invoke / run / execute / generate / analyze 这类方法，请解释它们的抽象层级
   - 谁是低层能力
   - 谁是高层流程
   - 为什么可能多个类都有同名方法

9. 请尽量使用一个生活比喻辅助理解，比如“用户点外卖的流程”
   - 用户问题 = 用户下单时写的需求
   - SQL = 商家后厨看到的小票
   - 数据库 = 商家的后厨/订单系统
   - QueryResult = 做好的餐品/出餐结果
   - GraphState = 一路流转的外卖订单
   - node = 接单、备餐、打包、配送等流程岗位
   - analyzer = 打包核对/客服解释
   - retry = 做错或信息不够时重新处理

10. 不要只翻译代码，要告诉我：
   - 正常程序员看到这里会优先关注什么
   - 哪些暂时不用深究
   - 如果后面出 bug，应该去哪个类/函数里查

11. 最后请用一句话总结这个文件：
   “这个文件本质上是在做什么？”

请用中文解释，尽量具体，结合代码片段，不要太抽象。
```
